# 视界伴音：视觉场景感知的上下文音乐推荐方案总结

## 1. 项目背景与需求
*   **传统推荐局限**：主要依赖用户历史行为，忽视了用户当前的物理环境和视觉情境，导致推荐结果与实际氛围脱节。
*   **场景化需求**：不同生活场景（如登山、咖啡馆、健身房）对音乐风格、节奏和能量有特定要求。
*   **技术支撑**：移动设备摄像头的普及以及 CLIP 等跨模态预训练模型的发展，为实时视觉场景理解提供了可能。

## 2. 问题定义
*   **输入**：移动设备实时捕获的图像数据（如自然风光、室内环境、街道等）。
*   **输出**：与当前视觉场景氛围、活动类型及情绪特征高度匹配的 Top-N 音乐推荐列表。

## 3. 系统架构
系统由以下四个核心模块组成：
1.  **视觉场景理解**：利用 **CLIP 编码器** 解析图像语义，提取 512 维视觉特征向量，识别场景属性。
2.  **音乐特征模块**：整合 **Spotify API** 与 **MSD (Million Song Dataset)**，提取音乐的情绪（Valence）、能量（Energy）、节奏（Tempo）等维度特征。
3.  **跨模态检索模块**：采用**双塔架构**，通过**余弦相似度**计算视觉特征与音乐特征的匹配度，并利用 **FAISS** 实现高效检索。
4.  **推荐排序模块**：融合场景权重、协同过滤及多样性策略，对候选歌曲进行精排。

## 4. 数据来源
*   **图像数据**：使用 **Places365** 数据集，涵盖 365+ 场景类别和 180 万张图像。
*   **音乐数据**：结合 MSD 的音频特征与 Spotify API 的声学属性（如情绪效价、能量感等）。
*   **标注数据**：构建场景-音乐匹配对，用于跨模态监督信号的微调与评估。

## 5. 核心创新点
*   **情境感知**：突破传统基于用户画像的推荐，开创由实时场景驱动的新模式。
*   **跨模态对齐**：实现视觉语义与音乐声学特征的深度对齐。
*   **轻量化部署**：系统架构支持在移动端或可穿戴设备（如智能眼镜）上实时运行。

## 6. 未来展望
*   **局限性**：需克服复杂光照对图像识别的影响，并扩展场景类别。
*   **扩展方向**：
    *   支持**视频流**实时动态推荐。
    *   引入**时序建模**，捕捉场景的连续变化。
    *   拓展至**车载、智能家居**及智能眼镜等多种应用场景。
